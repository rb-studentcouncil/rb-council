# คู่มือระบบปฏิทิน — สภานักเรียน RB

หน้า "ปฏิทิน" ดึงข้อมูลกิจกรรมจาก **Google Sheet 2 ชีต** (แยกหมวด)
สภาแก้ Sheet → เว็บอัปเดตเองในไม่กี่นาที (ไม่ต้อง push git)

ภาพรวม:
```
ชีต TCAS               ชีต กิจกรรมโรงเรียน
      │                          │
      │   (ทั้งคู่ publish เป็น CSV)
      ▼                          ▼
หน้า "ปฏิทิน" บนเว็บ — คอลัมน์ซ้าย=TCAS, ขวา=กิจกรรมโรงเรียน
```

---

## ขั้นที่ 1 — สร้าง Google Sheet

ไปที่ https://sheets.google.com สร้างไฟล์ใหม่ ตั้งชื่อ เช่น "RB Calendar"

**ในไฟล์เดียวกันให้มี 2 ชีต (tab ล่าง):**
- ชีต 1 ตั้งชื่อ `TCAS` (สำหรับ TCAS / สอบ / ทุน)
- ชีต 2 ตั้งชื่อ `School` (สำหรับกิจกรรมโรงเรียน)

> หรือจะสร้างเป็น 2 ไฟล์แยกก็ได้ — ระบบใช้ลิงก์ CSV คนละลิงก์อยู่แล้ว

### หัวคอลัมน์ (เหมือนกันทั้ง 2 ชีต):

ใส่หัวคอลัมน์แถวแรก (ตัวเล็กทั้งหมด สะกดตรงเป๊ะ):

```
title | date | date_iso | location | status | when_label
```

> ลำดับคอลัมน์สลับได้ ระบบอ่านตามชื่อหัว ไม่ใช่ตำแหน่ง

### ความหมายแต่ละคอลัมน์:

| คอลัมน์ | บังคับ | ตัวอย่าง | อธิบาย |
|--------|--------|----------|--------|
| `title` | ✅ | "ประกาศผล TCAS รอบ 3" | ชื่อกิจกรรม |
| `date` | ❌ | "26 พ.ค. 2569" หรือ "28 พ.ค. – 14 มิ.ย. 2569" | วันที่แบบไทย (ใช้แสดงในเว็บ) |
| `date_iso` | ❌ | `2026-05-26` หรือ `2026-05-28~2026-06-14` | วันที่แบบ ค.ศ. (ไว้ให้ระบบคำนวณแม่นๆ — ถ้าใส่ `date` ไทยก็พอแล้ว) |
| `location` | ❌ | "หอประชุมโรงเรียน" | สถานที่ (ถ้ามี) |
| `status` | ❌ | `soon`, `past`, หรือเว้นว่าง | บังคับสถานะ (ปกติให้ระบบคำนวณเอง) |
| `when_label` | ❌ | "อีก 4 วัน" | บังคับข้อความใน pill (ปกติให้ระบบคำนวณเอง) |

### ✨ ระบบคำนวณ status / pill อัตโนมัติ:

ถ้าคุณกรอกแค่ `title` + `date` ระบบจะคำนวณ pill อัตโนมัติให้:

- **กิจกรรมในอนาคต ≤30 วัน** → pill ทอง "อีก 5 วัน" / "พรุ่งนี้" / "วันนี้"
- **ช่วงวันที่กำลังเปิด** เช่น "28 พ.ค. – 14 มิ.ย." วันนี้คือ 1 มิ.ย. → pill ทอง "กำลังเปิด · เหลือ 13 วัน"
- **กิจกรรมในอนาคต >30 วัน** → pill ชมพู "อีก 45 วัน"
- **กิจกรรมที่ผ่านไปแล้ว** → pill เทา "ผ่านมา 3 วัน" (เรียงล่างสุด)

ถ้าอยาก override ระบบอัตโนมัติ ก็ใส่ `status` และ/หรือ `when_label` ของคุณเอง

### ค่า `status` ที่รองรับ (ใช้ override เท่านั้น):

- `soon` หรือ `เร็ว ๆ นี้` หรือ `ใกล้` → pill สีทอง
- `past` หรือ `ผ่านแล้ว` หรือ `ผ่าน` → pill สีเทา
- เว้นว่าง → ให้ระบบคำนวณเอง

### ตัวอย่างข้อมูล:

## ขั้นที่ 2 — publish ทั้ง 2 ชีตเป็น CSV

ทำขั้นตอนนี้กับ **ทั้ง 2 ชีต** (แยกกัน):

1. File → Share → Publish to web
2. ช่องซ้าย: เลือก **ชีตที่ต้องการ** (TCAS หรือ School ทีละอัน)
3. ช่องขวา: เลือก **Comma-separated values (.csv)**
4. กด Publish แล้ว copy ลิงก์ (หน้าตา `...pub?gid=...&single=true&output=csv`)

จะได้ลิงก์ 2 อัน — อันหนึ่งของชีต TCAS อันหนึ่งของชีต School
(สังเกตว่า `gid=` จะต่างกัน)

---

## ขั้นที่ 3 — ใส่ลิงก์ CSV ลงในเว็บ

1. เปิดไฟล์ `calendar.html`
2. หาบรรทัด:
   ```js
   const TCAS_CSV_URL   = '';
   const SCHOOL_CSV_URL = '';
   ```
3. วางลิงก์ CSV ที่ตรงกัน:
   ```js
   const TCAS_CSV_URL   = 'https://docs.google.com/spreadsheets/d/e/2PACX-xxxx/pub?gid=0&single=true&output=csv';
   const SCHOOL_CSV_URL = 'https://docs.google.com/spreadsheets/d/e/2PACX-xxxx/pub?gid=12345&single=true&output=csv';
   ```
4. เซฟ แล้ว push ขึ้น GitHub:
   ```bash
   git add .
   git commit -m "Connect calendar to Google Sheets"
   git push
   ```

> ใส่แค่ลิงก์เดียวก่อนก็ได้ — อีกคอลัมน์จะขึ้น "ยังไม่ได้ตั้ง Google Sheet"

## ใช้งานหลังจากนี้

- **เพิ่มกิจกรรมใหม่:** ไปที่ Sheet เลือกชีตที่ตรงหมวด เพิ่มแถว → เว็บอัปเดตเองใน 1-5 นาที
- **กิจกรรมผ่านไปแล้ว:** เปลี่ยน `status` เป็น `past` → ขึ้นไปอยู่ล่างสุด สีเทา
- **กิจกรรมใกล้ถึง:** เปลี่ยน `status` เป็น `soon` + ใส่ `when_label` "อีก X วัน"
- **ลบกิจกรรม:** ลบแถวใน Sheet
- ไม่ต้อง push git อีกหลังจากตั้ง URL แล้ว — แก้แค่ Sheet พอ

---

## ข้อควรรู้

- Google จะ cache ลิงก์ CSV ราว 1-5 นาที ถ้าแก้แล้วยังไม่เปลี่ยน รอสักครู่ + hard refresh (Cmd+Shift+R)
- Sheet ที่ publish จะเป็นสาธารณะ (ใครมีลิงก์ก็เห็น) — ใส่เฉพาะข้อมูลที่เผยแพร่ได้
- ถ้าดึง CSV ไม่ได้ ชีตนั้นจะขึ้น error box ส่วนชีตอีกอันยังทำงานปกติ
- ทำผิดชีต? แค่ลาก-วาง row ไปยังชีตที่ถูกได้เลย
