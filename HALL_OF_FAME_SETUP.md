# คู่มือระบบ Hall of Fame — สภานักเรียน RB

หน้า Hall of Fame ทำงานเป็นระบบครบวงจร:
**คนเสนอชื่อผ่าน Google Form → สภาตรวจ → อันที่ผ่านขึ้นเว็บอัตโนมัติ**

ภาพรวม:
```
คนกรอก Google Form (แนบหลักฐานได้)
      │  (อัตโนมัติ)
      ▼
ชีต "Form Responses 1"  ← คำตอบดิบ + ลิงก์ไฟล์หลักฐาน  (ยังไม่ขึ้นเว็บ)
      │  (สภา copy เฉพาะอันที่ตรวจผ่าน)
      ▼
ชีต "Published"  ← เฉพาะอันที่อนุมัติแล้ว  = ตัวที่เว็บดึงไปแสดง
      │  (publish เป็น CSV)
      ▼
หน้า Hall of Fame บนเว็บ
```

หัวใจของ "ตรวจก่อนเผยแพร่": **เว็บดึงจากชีต "Published" เท่านั้น**
คนกรอกฟอร์มเขียนได้แค่ชีต "Form Responses" จึงไม่มีทางขึ้นเว็บเองโดยไม่ผ่านสภา

ถ้ายังไม่ได้ตั้งค่าครบ เว็บจะใช้ข้อมูลตัวอย่างจาก `data/hall.json` ไปก่อน

---

## ขั้นที่ 1 — สร้าง Google Form

ไปที่ https://forms.google.com สร้างฟอร์มใหม่ ใส่คำถามตามนี้
(เรียงลำดับให้ตรงกับคอลัมน์ในชีต Published):

| # | คำถาม | ชนิด | บังคับ |
|---|-------|------|--------|
| 1 | ชื่อ–สกุล / ชื่อทีม | Short answer | ✅ |
| 2 | ระดับชั้น/ห้อง (ทีมเว้นว่างได้) | Short answer | ❌ |
| 3 | หมวด | Multiple choice: วิชาการ / กีฬา / ศิลปะ/ดนตรี / กิจกรรม | ✅ |
| 4 | รางวัล/ผลงาน | Short answer | ✅ |
| 5 | ปี (พ.ศ.) | Short answer | ❌ |
| 6 | Instagram (ถ้ามี) | Short answer | ❌ |
| 7 | รายละเอียดเพิ่มเติม | Paragraph | ❌ |
| 8 | แนบหลักฐาน (เกียรติบัตร/รูปถ่าย) | File upload | ✅ |

หมายเหตุ:
- หมวด ต้องเป็น Multiple choice เพื่อบังคับเลือกจากตัวเลือกตายตัว
  (กันพิมพ์ผิดแล้วกลายเป็นคนละหมวด)
- File upload ตั้งให้รับเฉพาะรูปภาพ + PDF, จำกัด 3-5 ไฟล์, ขนาด ~10MB
  - คนกรอกต้องล็อกอิน Google account (นักเรียนใช้บัญชีโรงเรียนได้)
  - ไฟล์เก็บใน Google Drive ของเจ้าของฟอร์ม ในชีตจะได้เป็น "ลิงก์" ไปยังไฟล์

ลิงก์ฟอร์มปัจจุบันที่ใส่ในเว็บแล้ว: https://forms.gle/x8GxQNVREQ3RCJjR7
(ถ้าสร้างฟอร์มใหม่ ให้แก้ลิงก์นี้ในไฟล์ hall-of-fame.html - หาคำว่า forms.gle)

## ขั้นที่ 2 — เชื่อมฟอร์มเข้า Google Sheet

ในฟอร์ม กดแท็บ Responses → ไอคอน Google Sheets สีเขียว (Link to Sheets)
→ Create new spreadsheet → ตั้งชื่อเช่น "Hall of Fame Data"

คำตอบจะไหลเข้าชีต "Form Responses 1" อัตโนมัติ (มีคอลัมน์ Timestamp
และหัวเป็นภาษาไทยตามคำถาม + คอลัมน์ลิงก์ไฟล์หลักฐาน)

---

## ขั้นที่ 3 — สร้างชีต "Published" (สำหรับอันที่ตรวจผ่าน)

ในไฟล์ Sheet เดียวกัน กด + ที่แท็บล่าง สร้างชีตใหม่ชื่อ Published
แถวแรกใส่หัวคอลัมน์ 8 ช่องนี้ (ตัวเล็กทั้งหมด สะกดตรงเป๊ะ):

```
name | class | category | award | year | igusername | detail | photo
```

> ลำดับคอลัมน์สลับได้ ระบบอ่านตามชื่อหัว ไม่ใช่ตำแหน่ง
> แต่ "ชื่อหัว" ต้องสะกดตรงตามนี้

**คอลัมน์ `photo`** ใส่รูปได้ 3 แบบ:
- **ชื่อไฟล์เฉยๆ**: `award-001.jpg` ← ระบบเติม `images/hall/` ให้อัตโนมัติ (ง่ายสุด)
- **path เต็ม**: `images/hall/award-001.jpg`
- **URL ภายนอก**: `https://example.com/image.jpg`
- ถ้าไม่มีรูป ปล่อยว่าง ระบบจะใช้ตัวอักษรแรกของชื่อรางวัลเป็น placeholder อัตโนมัติ

ก่อนใช้: อัปโหลดไฟล์รูปขึ้นโฟลเดอร์ `images/hall/` ใน repo ก่อน (push ขึ้น GitHub)

---

## ขั้นที่ 4 — ขั้นตอนตรวจของสภา (ทำทุกครั้งที่มีคนเสนอ)

1. เปิดชีต "Form Responses 1" ดูรายการใหม่
2. กดลิงก์ในคอลัมน์ไฟล์ เพื่อเปิดดูหลักฐาน (เกียรติบัตร/รูปถ่าย)
3. ถ้าข้อมูลถูกต้องและหลักฐานน่าเชื่อถือ → copy ข้อมูลไปวางในชีต Published
   - วางให้ตรงคอลัมน์: name, class, category, award, year, igusername, detail
   - ไม่ต้อง copy คอลัมน์ Timestamp และคอลัมน์ไฟล์ (เว็บไม่ใช้)
4. อันที่ไม่ผ่าน → ไม่ต้อง copy (ก็จะไม่ขึ้นเว็บ)

> เคล็ดลับ: จะเพิ่มคอลัมน์ "ตรวจแล้ว" ในชีต Form Responses ไว้กันสับสน
> ว่าอันไหนพิจารณาไปแล้วก็ได้

---

## ขั้นที่ 5 — publish ชีต "Published" เป็น CSV

1. File → Share → Publish to web
2. ช่องซ้าย: เลือกชีต Published (ห้ามเลือก Form Responses หรือ Entire Document)
3. ช่องขวา: เลือก Comma-separated values (.csv)
4. กด Publish แล้ว copy ลิงก์ (หน้าตาเหมือน ...pub?gid=...&single=true&output=csv)

---

## ขั้นที่ 6 — ใส่ลิงก์ CSV ลงในเว็บ

1. เปิดไฟล์ hall-of-fame.html
2. หาบรรทัด:
   ```js
   const SHEET_CSV_URL = '';
   ```
3. วางลิงก์ CSV ลงไป:
   ```js
   const SHEET_CSV_URL = 'https://docs.google.com/spreadsheets/d/e/2PACX-xxxx/pub?gid=...&single=true&output=csv';
   ```
4. เซฟ แล้ว push ขึ้น GitHub:
   ```bash
   git add .
   git commit -m "Connect Hall of Fame to Published sheet"
   git push
   ```

เสร็จแล้ว! การ์ดบนเว็บจะมาจากชีต Published ที่สภาคุมเอง

---

## ใช้งานหลังจากนี้

- มีคนเสนอชื่อใหม่ → ทำขั้นที่ 4 (ตรวจ + copy ไป Published) → เว็บอัปเดตเองใน
  ไม่กี่นาที
- แก้/ลบรายชื่อบนเว็บ → แก้ในชีต Published ได้เลย
- เพิ่มหมวดใหม่ → พิมพ์ชื่อหมวดใหม่ในคอลัมน์ category ปุ่มกรองจะขึ้นเอง
  (อย่าลืมเพิ่มตัวเลือกหมวดในฟอร์มด้วย)

## ข้อควรรู้

- หน้านี้ต้องเปิดผ่านโฮสต์จริง (GitHub Pages) - เปิดไฟล์ตรง ๆ (file://) ไม่ได้
- ชีต Published ที่ publish จะเป็นสาธารณะ (ใครมีลิงก์ก็เห็น)
  → ใส่เฉพาะข้อมูลที่เผยแพร่ได้ ส่วนหลักฐาน/ข้อมูลส่วนตัวเก็บไว้ใน Form Responses
- รูปหลักฐานจาก Form ไม่ขึ้นบนเว็บ มีไว้ให้สภาดูตอนตรวจเท่านั้น
- ถ้าดึง CSV ไม่ได้ เว็บจะถอยกลับไปใช้ data/hall.json อัตโนมัติ
- Google cache ลิงก์ CSV ราว ๆ ไม่กี่นาที ถ้าแก้แล้วยังไม่เปลี่ยน รอสักครู่
  แล้ว refresh แบบล้าง cache (Ctrl+Shift+R)
